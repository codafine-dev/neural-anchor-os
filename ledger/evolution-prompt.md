# System Evolution Ledger Prompt

## Goal
Capture a "System Shift" in the Neural-Anchor OS to prevent cognitive loss during architectural changes.

## Instructions
When a change is made to the way knowledge is captured, stored, or retrieved, record it using the following structure:

1. **The Shift**: What exactly changed? (From X $\rightarrow$ Y)
2. **The Trigger**: Why was this change necessary? (e.g., "Brain can't move", "Too much friction in X")
3. **The New Logic**: How does the new system work?
4. **Impact**: What does this change for future operations?

## Output Format
- **Date**: {{date}}
- **Change Type**: [Architecture | Tooling | Workflow]
- **Shift**: {{old_state}} ➡️ {{new_state}}
- **Reasoning**: {{the_why}}
- **Instruction**: {{how_to_use_now}}
