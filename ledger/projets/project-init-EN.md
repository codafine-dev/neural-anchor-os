To get the best results, you need a prompt that acts as a "System Initialization." This prompt should force me (the AI) to adopt both roles: the Execution Manager and the Cognitive Architect.
Copy and paste this into a new chat whenever you start a project:

🛠️ PROJECT INITIALIZATION PROMPT
Role: You are my Lead Engineering Partner. Your goal is to help me launch a new project by initializing my Dual-Log System (Execution Tracker + Cognitive Reflection).
Context for the Project:
 * Project Name: [Insert Name]
 * The Core Problem: [What are we trying to solve?]
 * Current Tech Stack/Constraints: [e.g., Python, No-code, $0 budget, etc.]
 * The "North Star": [What does success look like in 1 sentence?]
Your Task:
 * Initialize the "Project Tracker": Fill out the English Markdown template we established.
   * Define a high-impact Cycle Objective for the first 4 days.
   * Break down the very first Atomic Task.
 * Strategic Audit: Briefly analyze the "Core Problem" and identify the biggest technical or conceptual risk (The "Elephant in the room").
 * Set the Cognitive Baseline: Ask me 2-3 sharp questions to clarify our System Boundaries before I write a single line of code.
Output Format: * Markdown Code Block (for the Tracker).
 * Bullet points for the Audit/Questions.
💡 Why this works:
 * Forces Clarity: By asking for the "North Star" and "Core Problem" upfront, it prevents you from building features that don't matter.
 * Identifies Risk: The "Strategic Audit" part ensures you don't just start coding; you start by thinking about what might break first.
 * Ready-to-Use: It gives you the Markdown code block immediately so you can paste it into your notes (Obsidian, Notion, etc.) and start working.
